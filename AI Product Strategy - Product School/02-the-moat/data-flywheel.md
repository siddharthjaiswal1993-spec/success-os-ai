# Data Flywheel Map — SuccessOS AI

Scored at ground zero (pre-build). The first flywheel is correction data, not network effects.

## Flywheel Loops

| Loop | What It Measures | Score |
|------|------------------|-------|
| Correction | Do users fix AI outputs, and is that signal captured? | 1/5 |
| Preference | Does it learn individual/team preferences over time? | 1/5 |
| Domain Context | Does usage in one area improve quality in others? | 2/5 |
| Network | Does each new user make it better for everyone? | 1/5 |

**Correction (1/5):** Approve/Dismiss exists but nothing is stored. Every override should become training data tied to account context and eventual outcome.

**Preference (1/5):** No memory of team behavior. Every session starts from zero.

**Domain Context (2/5):** Shared account data exists, but relationships are rule-based rather than learned.

**Network (1/5):** No cross-customer learning today. Future path is anonymized benchmarks and intervention outcome patterns.

**Total: 5/20**
**Weakest loop:** Correction.

## Encroachment Threats

| Attacker | Vector | Time to Threat | Value at Risk |
|----------|--------|----------------|---------------|
| Salesforce / Einstein | Native CRM intelligence and workflow automation | 12–18 mo | 60–70% |
| Gainsight / ChurnZero | AI layered onto existing CS workflows | In market now | 50–60% |
| Gong | Call intelligence expanding into CS | 18–24 mo | 30–40% |

**90-day red team:** Salesforce ships native churn insights inside the CRM.

**Our defense:** Salesforce optimizes CRM workflows. SuccessOS optimizes customer outcome workflows. The moat is not health scoring; it is intervention prioritization, human approval, cross-system intelligence, and accumulated correction data.
