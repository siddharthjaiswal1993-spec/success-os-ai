# Data Model — SuccessOS AI

---

## Core Entities

### Account
The central entity. Everything in the system is organized around accounts.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| tenant_id | UUID | Multi-tenant isolation |
| name | String | Customer account name |
| arr | Integer | Annual recurring revenue (USD cents) |
| renewal_date | Date | Next renewal date |
| csm_owner_id | UUID | FK → User |
| health_score | Integer (0–100) | Composite AI health score |
| health_tier | Enum | Red / Yellow / Green |
| products_used | String[] | Product lines active |
| segment | Enum | SMB / Mid-Market / Enterprise |
| industry | String | Customer industry vertical |
| created_at | Timestamp | |
| updated_at | Timestamp | |

---

### AccountSignal
A single signal event that affects account health or triggers agent analysis.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| account_id | UUID | FK → Account |
| signal_type | Enum | meeting / support_ticket / usage_event / health_change / expansion_signal / risk_flag |
| signal_date | Date | When the signal occurred |
| summary | Text | Plain-language description |
| sentiment | Enum | Very Positive / Positive / Neutral / Cautious / Negative |
| severity | Enum | Critical / High / Medium / Low / Info |
| raw_content_ref | UUID | Optional FK → raw content store |
| created_by | Enum | ai_agent / csm_manual / integration_sync |

---

### HealthScoreHistory
Immutable log of health score changes for trend analysis.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| account_id | UUID | FK → Account |
| score | Integer | Score at this point in time |
| tier | Enum | |
| component_scores | JSONB | Adoption/Sentiment/Support/Engagement/Renewal sub-scores |
| change_reason | Text | Why the score changed |
| recorded_at | Timestamp | |

### RiskFlag
A structured, actionable risk associated with an account.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| account_id | UUID | FK → Account |
| risk_type | Enum | Adoption Decline / Support Issues / Engagement Gap / Renewal Risk / Sentiment Decline / Stakeholder Change |
| description | Text | What the risk is |
| severity | Enum | Critical / High / Medium |
| evidence_signal_ids | UUID[] | FK[] → AccountSignal |
| status | Enum | Open / In Progress / Resolved / Dismissed |
| opened_at | Timestamp | |
| resolved_at | Timestamp | Nullable |

---

### ExpansionSignal
A detected expansion opportunity for an account.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| account_id | UUID | FK → Account |
| opportunity_type | Enum | Team Expansion / New Use Case / Budget Signal / Feature Request / Exec Interest |
| description | Text | What was detected |
| source_signal_id | UUID | FK → AccountSignal |
| opportunity_score | Integer (0–100) | AI-assessed opportunity strength |
| status | Enum | Active / Actioned / Expired |
| detected_at | Timestamp | |
| expires_at | Timestamp | Auto-expire after 45 days if not actioned |

### AgentAction
A recommended action generated by an AI agent.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| account_id | UUID | FK → Account |
| action_type | Enum | Executive Check-in / Expansion Outreach / Health Score Update / Risk Flag / QBR Generation / Manager Escalation / Renewal Discussion |
| boundary | Enum | Internal / External |
| priority | Enum | Critical / High / Medium |
| reasoning | Text | Why the AI recommended this action |
| draft_content | Text | Draft email / message / note |
| confidence_score | Float (0–1) | AI confidence in this recommendation |
| status | Enum | Pending / Approved / Dismissed / Auto-executed |
| approver_id | UUID | FK → User (nullable) |
| dismiss_reason | Enum | Not relevant / Already handled / Timing / Quality / Other |
| created_at | Timestamp | |
| resolved_at | Timestamp | Nullable |

### FeatureImpactAnalysis
A feature-to-account mapping analysis.

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | |
| feature_description | Text | Input from user |
| created_by | UUID | FK → User |
| created_at | Timestamp | |

## Relationship Summary

```
Tenant
  └── Users (CSMs, Directors, Admins)
  └── Accounts
        ├── AccountSignals (timeline events)
        ├── HealthScoreHistory (immutable audit trail)
        ├── RiskFlags (active risks)
        ├── ExpansionSignals (opportunities)
        ├── AgentActions (AI recommendations)
        └── QBRBriefs (generated documents)
```
